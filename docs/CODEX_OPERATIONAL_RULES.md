# Codex Operational Rules

## Objetivo
Impedir criacao indevida de estruturas temporarias, archived, backup, recovery, clones ou copias paralelas fora da governanca oficial.

## Regras operacionais imediatas
1. Nao criar pastas temporarias sem autorizacao explicita.
2. Nao criar pastas `archived` sem autorizacao explicita.
3. Nao criar backups novos sem autorizacao explicita.
4. Nao criar `recovery` novo sem autorizacao explicita, exceto quando houver risco real de perda e o Owner autorizar.
5. Nao criar clones, copias paralelas ou pastas alternativas.
6. Se uma pasta temporaria, archived, backup ou recovery for criada por engano, remover imediatamente quando solicitado, sem insistir.
7. Antes de qualquer limpeza, verificar se o caminho e oficial, duplicado, cache, build ou conteudo unico.

## Fontes oficiais
- GitHub `main` = fonte oficial.
- `D:\AI-constr\AI-Construction-Intelligence-Platform` = workspace oficial.
- `D:\AI-constr\EBOOK_APEX_HOTMART` = ebook.
- `D:\AI-constr\SKILLS_APEX` = skills.
- `D:\AI Jedgard` = acervo, Revit e material original.

## Interpretacao operacional
- Qualquer criacao de pasta fora do workspace oficial precisa de autorizacao explicita do Owner.
- `temp`, `tmp`, `archived`, `backup`, `recovery`, clones e variantes equivalentes entram no bloqueio preventivo.
- Cache e artefatos de build so podem ser tratados como descartaveis depois da verificacao de caminho e natureza do conteudo.
- Quando houver duvida entre conteudo unico e artefato regeneravel, assumir postura conservadora e nao criar nem limpar sem autorizacao.
